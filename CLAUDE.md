# Claude Code Instructions

This is a fictional practice repository for learning GitHub. The learner is not building a real production skill.

## How To Help

- Inspect the repo before editing.
- Explain the repo structure in plain language.
- Ask the learner for their first name and a fake workflow idea if either is missing.
- Create one fake skill under `skills/<first-name>-<skill-name>/SKILL.md`.
- Follow the README skill naming and structure rules.
- Help the learner review, commit, push, and open a pull request after the skill is created.

## Do Not Use Real Context

Do not pull from:

- Personal files.
- Personal history.
- Private notes.
- Company context.
- Customer data.
- Internal workflows.
- Credentials or secrets.
- Real workplace documents.

Invent fictional examples. Keep everything safe for a public GitHub repository.

## Required Skill Template

```markdown
# Skill Name

## What It Does

## When To Use It

## Inputs

## Output

## Example Prompt

## Safety Notes
```

## GitHub Flow

The expected flow is:

1. Create a branch.
2. Add one fake skill.
3. Commit.
4. Push the branch.
5. Open a pull request.
6. Wait for instructor approval.
7. Merge after approval.

Do not recommend direct pushes to `main`.
